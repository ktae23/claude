# Fix Tests

테스트를 실행하고 실패하는 테스트를 수정합니다. 프로덕션 코드는 절대 수정하지 않고 테스트 코드만 수정합니다.

## Instructions

다음 단계를 테스트가 모두 통과할 때까지 반복합니다:

### 1. 테스트 실행

프로젝트의 테스트를 실행합니다:

```bash
./mvnw test -pl api -DfailIfNoTests=false 2>&1 | tail -500
```

### 2. 결과 분석

- 모든 테스트가 통과하면 사용자에게 성공을 알리고 종료합니다
- 실패한 테스트가 있으면 다음 단계로 진행합니다

### 3. 실패 원인 파악

실패 로그를 분석하여:
- 어떤 테스트 클래스/메서드가 실패했는지 확인
- 실패 원인 (assertion 실패, 예외 발생 등) 파악
- 예상값과 실제값 비교

### 4. 테스트 코드 수정

**중요: 프로덕션 코드(`src/main/`)는 절대 수정하지 않습니다. 테스트 코드(`src/test/`)만 수정합니다.**

실패 원인에 따라 테스트 코드를 수정합니다:
- 예상값이 잘못된 경우: 올바른 예상값으로 수정
- Mock 설정이 잘못된 경우: Mock 동작 수정
- 테스트 데이터가 잘못된 경우: 테스트 데이터 수정
- 테스트 로직이 현재 구현과 맞지 않는 경우: 테스트 로직 수정

### 5. 반복

1단계로 돌아가 테스트를 다시 실행합니다. 모든 테스트가 통과할 때까지 반복합니다.

## Notes

- 프로덕션 코드 수정 금지: `src/main/` 하위 파일은 읽기만 가능, 수정 불가
- 테스트 코드만 수정: `src/test/` 하위 파일만 수정 가능
- 최대 10회 반복 후에도 실패하면 사용자에게 보고하고 종료
- 각 반복마다 어떤 테스트가 실패했고 어떻게 수정했는지 간략히 보고
